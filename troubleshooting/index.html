Access Issues¶

Can't Access Etherpad Plus¶

Symptoms: - Browser shows "connection refused" - Page won't load - Timeout errors

Solutions:

  1. Check if services are running: bash docker compose ps
  2. Services should show "Up" status

  3. If not running, start with: docker compose up -d

  4. Check port configuration:

  5. Verify port in .env: DOCKER_COMPOSE_APP_PORT_PUBLISHED
  6. Default: 11155

  7. Try: http://localhost:11155

  8. Check firewall:

  9. Ensure port is not blocked
  10. Check firewall rules

  11. Verify network access

  12. Check logs: bash docker compose logs etherpad

  13. Look for error messages
  14. Check startup errors

Wrong Port or URL¶

Problem: Can't find Etherpad at expected URL

Solutions: - Check .env file for port configuration - Verify domain/FQDN setting - Check reverse proxy configuration (if used) - Try default: http://localhost:11155

Solutions:

  3. Refresh page if disconnected

  4. Check network connection

  6. Try different browser
  7. Clear browser cache

  8. Disable browser extensions

  9. Check database: bash docker compose logs postgres

  10. Look for database errors
  11. Verify database is running

Solutions:

  2. Verify ep_author_neat2 is installed

  3. Check plugin status in admin panel

  4. Refresh page:

  5. Reload browser page

  6. Clear browser cache

  7. Check network:

  8. Verify WebSocket connection
  9. Check firewall for WebSocket ports

  7. Try keyboard shortcuts:

  9. Ctrl+I for italic

  12. Reload browser

Formatting Issues¶

Headings Not Working¶

Symptoms: - Heading dropdown doesn't work - Headings don't appear different - TOC not generating

Solutions:

  1. Check plugin:
  2. Verify ep_headings2 installed

  3. Check ep_table_of_contents for TOC

  4. Use dropdown:

  5. Select heading level from dropdown

  6. Not just typing "#" symbol

  7. Check formatting:

  8. Headings may look similar
  9. Check in exported document

Images Not Uploading¶

Symptoms: - Upload button doesn't work - Images don't appear - Upload fails

Solutions:

  1. Check file size:
  2. Maximum: 5MB per image

  3. Compress large images

  4. Check file format:

  5. Supported: JPEG, PNG, GIF, BMP

  6. Convert if needed

  7. Check permissions:

  8. Verify image directory writable

  9. Check container permissions

  10. Check plugin:

  11. Verify ep_image_upload installed
  12. Check plugin configuration

Solutions:

  2. Must select text before coloring

  3. Empty selection won't work

  4. Check plugin:

  5. Verify ep_font_color installed

  6. Check plugin status

  7. Try different browser:

  8. Browser compatibility issue
  9. Update browser

Export Problems¶

Export Button Missing¶

Symptoms: - Can't find export option - File menu empty - No export button

Solutions:

  1. Check menu:
  2. Look in File menu
  3. May be in different location

  4. Check toolbar

  5. Check permissions:

  6. Export may require login

  7. Check authentication settings

  8. Use API:

  9. Try direct URL: /p/PadName/export/html
  10. Or /p/PadName/export/odt

Export File Corrupted¶

Symptoms: - File won't open - Error opening file - File appears empty

Solutions:

  1. Re-export:
  2. Try exporting again

  3. May be temporary issue

  4. Try different format:

  5. Try HTML instead of ODT

  6. Or ODT instead of DOCX

  7. Check LibreOffice:

  8. DOCX requires LibreOffice
  9. Verify LibreOffice installed
  10. Check server logs

Images Missing in Export¶

  2. Verify supported formats

  3. Re-upload if needed

  4. Check image size:

  5. Large images may fail

  6. Optimize images

  7. Check export format:

  8. Some formats handle images differently
  9. Try HTML format

Performance Issues¶

Slow Loading¶

Symptoms: - Page takes long to load - Laggy editing - Timeouts

Solutions:

  1. Check server resources: bash docker stats
  2. Monitor CPU/memory usage

  3. Add resources if needed

  4. Check document size:

  5. Large documents load slowly

  6. Consider breaking into sections

  7. Check images:

  8. Many/large images slow loading
  9. Optimize images

  10. Remove unnecessary images

  11. Check database: bash docker compose logs postgres

  12. Look for slow queries
  13. Optimize database

Laggy Editing¶

Symptoms: - Typing delay - Cursor lag - Slow response

Solutions:

  1. Check connection:
  2. Slow network causes lag
  3. Check internet connection

  4. Try different network

  5. Reduce document size:

  6. Large documents lag

  7. Break into smaller pads

  8. Close other tabs:

  9. Browser resources limited

  10. Close unnecessary tabs

  11. Check browser:

  12. Update browser
  13. Clear cache
  14. Try different browser

High Memory Usage¶

Symptoms: - Server slow - Out of memory errors - Crashes

Solutions:

  1. Check memory: bash docker stats
  2. Monitor memory usage

  3. Add memory if needed

  4. Clean up pads:

  5. Delete unused pads

  6. Archive old documents

  7. Optimize database: bash docker compose exec postgres psql -U etherpad -d etherpad -c "VACUUM;"

  8. Restart services: bash docker compose restart

  10. Check for updates
  11. Update to latest versions
  12. May fix conflicts

  2. Verify .env file password
  3. Check for typos

  4. Reset if needed

  5. Check configuration:

  6. Verify admin user configured
  7. Check config/etherpad.settings.json

  8. Review authentication settings

  9. Reset password:

  10. Update .env file
  11. Restart services
  12. Try again

Database Connection Failed¶

Symptoms: - "Database error" messages - Can't save changes - Service won't start

Solutions:

  1. Check database: bash docker compose ps postgres docker compose logs postgres
  2. Verify database running

  3. Check for errors

  4. Check credentials:

  5. Verify .env database settings
  6. Check username/password

  7. Test connection

  8. Restart database: bash docker compose restart postgres

  9. Check network:

  10. Verify services on same network
  11. Check compose.yml network config

Services Won't Start¶

Symptoms: - Docker containers exit - Services crash - Startup errors

Solutions:

  1. Check logs: bash docker compose logs
  2. Review all error messages

  3. Identify root cause

  4. Check configuration:

  5. Verify .env file correct
  6. Check compose.yml syntax

  7. Review settings file

  8. Check resources: bash docker system df

  9. Verify disk space

  10. Check memory available

  11. Rebuild: bash docker compose down docker compose build --no-cache docker compose up -d

Common Error Messages¶

"Connection Lost"¶

Meaning: Lost connection to server

Fix: - Refresh page - Check network connection - Verify services running

"Database Error"¶

Meaning: Database connection issue

Fix: - Check database service - Verify credentials - Check logs

"Permission Denied"¶

Meaning: Access denied

Fix: - Check authentication - Verify permissions - Contact administrator

"Plugin Not Found"¶

Meaning: Plugin not installed

Fix: - Check plugin installation - Rebuild image - Verify plugin name

Still having issues? Contact your system administrator with: - Error messages - Steps to reproduce - Log output - Browser/OS information
